# Vibe Hosting / Vibe Apps - ビジネスコンセプト

## サービスのビジョン・思想
- 経営者・事業者はコードではなくビジネス成長に集中すべきという思想。
- LP 止まりの AI サイトビルダーではなく、業務が回るフルスタック Web アプリを自動生成。
- NO CODE ではなく NO THINKING に近い体験を目指し、自然言語の会話だけで要件定義からデプロイまで完結。

## 解決したい課題
- 中小企業・個人事業主が抱える課題
  - エンジニア採用・外注のコスト負担とコミュニケーションコスト。
  - ノーコードツールの学習・運用コスト、運用担当の確保難。
  - LP だけでは売上や業務効率化に直結しない。
- 雑なプロンプトから生成されるスパゲッティコードが「動けばいい」で放置される現状。
- 仕様が曖昧でも業務システムに落としてくれる「AI エンジニア」の不足。

## ターゲットユーザ・ペルソナ
- 地方の青果店 2 代目オーナー：在庫・受注管理と EC を統合したいがエンジニア不在。
- 個人輸入 EC オーナー：決済・配送連携を自動化し、管理コストを下げたい。
- 美容室経営者：予約と顧客管理を一元化しつつ、スタッフ別の権限管理も必要。
- 不動産仲介業者：物件一覧と問い合わせ動線をオンライン化したいがノーコードでは限界。

## 価値提案（Value Proposition）
- LP ではなく業務で使える Next.js フルスタック Web アプリを AI が生成。
- ソースコードの所有（買い切り/サブスク＋ライセンス）により、将来のエンジニアへの引き渡しが容易。
- 要件ヒアリングからデプロイ、プレビュー提供までワンストップ。
- サーバサイド処理、DB 設計、認証、決済まで含めて自動化し、ビジネス立ち上げ速度を最大化。

## 競合・代替手段との比較
| 項目 | Vibe Hosting / Vibe Apps | AI サイトビルダー (Framer AI 等) | 受託開発 | ノーコード (Bubble 等) |
| --- | --- | --- | --- | --- |
| 生成物 | 業務対応フルスタック Web アプリ | LP/簡易サイト中心 | カスタム開発 | 画面中心アプリ |
| コード引き渡し | あり（所有権確保） | 限定/不可 | あり | なし/制限多い |
| 要件定義支援 | AI が自動ヒアリング | 簡易フォーム | 人が対応 | 利用者が自力 |
| デプロイ | Vercel/CF Pages 自動 | ホスティング限定 | 案件ごと | プラットフォーム内 |
| 拡張性 | Next.js/TS ベースで高い | 低〜中 | 高 | プラットフォーム依存 |

## ビジネスモデル
- 生成＋ホスティングのサブスクリプション（プレビュー提供・軽微な改修込み）。
- ソースコード買い切りプラン（小〜中規模案件向け、引き渡し後は自由に改修可能）。
- オプション：カスタマイズ開発、要件コンサルティング、テンプレート販売。
- 初期フェーズは「生成回数課金＋ホスティング定額」で実需とフィードバックを素早く獲得。

## 将来の拡張可能性・プロダクトライン
- Vibe Studio：生成後の管理画面・運用ダッシュボード。
- Vibe Deploy：マルチクラウド/マルチリージョンデプロイの自動最適化。
- テンプレートギャラリー：業種別スターターと再利用可能なモジュール販売。
- Vibe Insights：運用データの可視化と改善提案を行うアナリティクス。

## リスク・懸念点と対策
- 法的リスク：商標・生成コードのライセンス確認、OSS ライセンス自動チェックを標準化。
- 技術的リスク：AI 生成品質と保守性を担保するため、Lint/テスト生成を自動組み込み、レビューエージェントを併用。
- ビジネスリスク：市場教育コストの高さ → 成功事例テンプレートを先に公開し、導入後の ROI を定量化。
- 運用リスク：デプロイ先依存性 → Vercel/Cloudflare Pages 両対応、DB 抽象化層を設けて移行性を確保。
